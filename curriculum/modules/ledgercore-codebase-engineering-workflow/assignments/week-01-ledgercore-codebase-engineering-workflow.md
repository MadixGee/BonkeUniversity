# Implement LedgerCore Transaction Logging System

Students will build a transaction logging system that records financial transactions to both a file and a database, simulating real-world engineering workflows for LedgerCore. This exercise reinforces integration between code, data persistence, and documentation practices.

## Deliverables
- A working script that logs transactions with timestamps, amounts, and statuses to a file and a database
- A README file explaining the logging workflow and setup instructions
- A configuration file for environment variables (e.g., database connection strings)

## Acceptance Criteria
- The script successfully writes transaction data to both a file and a database with matching timestamps and details
- The database contains at least 5 test transactions with verifiable entries
- The README includes clear instructions for running the script and database setup
- The configuration file correctly uses environment variables for database credentials
- All code is properly documented with comments explaining key engineering decisions
