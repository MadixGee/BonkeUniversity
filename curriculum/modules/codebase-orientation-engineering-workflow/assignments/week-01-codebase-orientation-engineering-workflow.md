# Environment Setup & The Developer Workflow

In this introductory assignment, you will set up a professional development environment and master the basic Git workflow used in industry. You will initialize a new project repository, manage feature branches, and document your setup process to ensure reproducibility.

## Deliverables
- A local Git repository containing a project structure with a README.md
- A.gitignore file configured for your specific programming language
- A series of Git commits following conventional commit standards
- A README.md file containing clear setup instructions

## Acceptance Criteria
- The repository must contain a valid.gitignore file that prevents build artifacts and environment secrets from being tracked.
- The project must demonstrate a 'feature branch' workflow where work is committed to a branch named 'feature/initial-setup' before being merged into 'ain'.
- Commit messages must follow the Conventional Commits specification (e.g., 'feat: initial project structure').
- The README.md must include a 'Prerequisites' section and a 'Getting Started' section that allows another developer to replicate your environment.
