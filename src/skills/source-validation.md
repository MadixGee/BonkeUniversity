# source-validation

Required workflow before drafting lesson content:

1. Identify 4-6 candidate primary sources for the topic:
- official documentation
- peer-reviewed papers
- well-established textbooks
- maintainer or creator statements
Use blog posts or aggregators only if no stronger primary source exists.

2. Verify each source:
- include a real URL or citation
- exclude unverifiable sources
- never guess or fabricate a source

3. Ground claims in sources:
- every factual claim in whyThisMatters, architectureDiscussion, and citedClaims must map to vetted sources

4. Keep an auditable resources trail:
- resources.md must list each source used with title, URL, and one-line rationale

5. Use research tools when available:
- verify source authenticity and version-specific details with web search or other research tools
- if uncertainty remains, flag it explicitly instead of asserting as fact
